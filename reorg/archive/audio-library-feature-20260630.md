### 音檔庫功能 — 新增總結 (2026-06-30 新增)

## 需求

系統原本各功能（IVR、分機、語音信箱）的音檔各自獨立上傳管理，沒有統一入口可以瀏覽、試聽、刪除所有自訂音檔，也看不到系統內建可用的提示音清單。新增獨立的「音檔庫」頁面，集中管理自訂音檔，並讓 IVR 等既有功能改為共用同一份檔案，避免重複實作。

## 功能說明

- 側邊欄新增「🔊 音檔庫」頁面，集中顯示：
  - **自訂音檔**：使用者上傳，存放於 `/var/lib/freeswitch/sounds/custom/`，可上傳／刪除／試聽
  - **系統內建音檔**：`/usr/share/freeswitch/sounds/` 下 `en`、`es`、`fr`、`pt`、`ru`、`music` 六個分類，遞迴掃描取得，唯讀僅可試聽
- 支援分類篩選（全部／自訂／各語系／保留音樂）
- 支援檔名關鍵字搜尋（debounce 300ms，避免輸入時頻繁重繪導致失焦）
- 內建音檔筆數多（實測 1016 筆），採分頁載入（預設 100 筆，可手動「載入更多」），避免一次渲染上千個 DOM 拖垮畫面
- 統一試聽播放器：清單每筆音檔只有一個播放／暫停按鈕，實際播放共用畫面下方同一個 `<audio>` 元件，按鈕圖示依播放器真實事件（`play`/`pause`/`ended`）同步，不會有按鈕顯示與實際播放狀態不一致的問題
- 刪除自訂音檔前會檢查是否被 IVR 引用中：
  - 未使用 → 直接刪除
  - 使用中 → 跳出警告列出引用的 IVR 名稱，需二次確認才能強制刪除（`force=true`）

## 修改檔案總覽

| 檔案 | 修改內容 |
|------|---------|
| server.py | 新增 `/api/sounds/*` 共用音檔庫 API；原 `/api/ivr/sounds/*` 四支端點改為委派呼叫，保留向下相容 |
| index.html | 新增音檔庫頁面（側邊欄、`pages` 註冊、`renderSoundLibrary()`）；IVR 編輯器內的音檔上傳/刪除/試聽改接同一組 API |

## 後端 API 端點

| Method | Endpoint | 說明 |
|--------|----------|------|
| `GET` | `/api/sounds/list?category=` | 列出音檔。category：`all`（預設）／`custom`／內建分類路徑 |
| `POST` | `/api/sounds/upload` | 上傳自訂音檔（格式、大小、重名檢查） |
| `DELETE` | `/api/sounds/{filename}?force=` | 刪除自訂音檔，預設先擋下使用中的檔案，`force=true` 強制刪除 |
| `GET` | `/api/sounds/usage?filename=` | 查詢該音檔目前被哪些 IVR 引用 |
| `GET` | `/api/sounds/stream?path=` | 串流播放（試聽用） |

舊端點 `/api/ivr/sounds/list`、`upload`、`{filename}`（DELETE）、`stream` 仍保留於 server.py，內部已改為呼叫上述共用函式，行為完全相容，前端已不再呼叫這組舊路徑。

## 技術細節

### 內建音檔掃描

```python
BUILTIN_SOUND_ROOTS = [
    ('/usr/share/freeswitch/sounds/en',    '英文提示音'),
    ('/usr/share/freeswitch/sounds/es',    '西班牙文提示音'),
    ('/usr/share/freeswitch/sounds/fr',    '法文提示音'),
    ('/usr/share/freeswitch/sounds/pt',    '葡萄牙文提示音'),
    ('/usr/share/freeswitch/sounds/ru',    '俄文提示音'),
    ('/usr/share/freeswitch/sounds/music', '保留音樂（MOH）'),
]
```

採用 `os.walk()` 遞迴掃描（深度上限 6 層），因為官方音檔包底下還會再分語者／取樣率／功能子目錄（如 `en/us/callie/ivr/8000/`），無法假設固定子路徑。每個分類最多列出 200 筆（`BUILTIN_LIST_LIMIT`），避免單次回應過大。

### 安全性

- 檔名白名單正則 `^[\w\-. ]+\.(wav|mp3|ogg|gsm)$`，防止路徑穿越
- 上傳限制副檔名與 20MB 大小上限
- 上傳重名直接拒絕（409），避免覆蓋既有檔案
- 串流／試聽路徑限制在白名單目錄（自訂音檔目錄＋內建音檔根目錄）內，防止任意檔案讀取
- 刪除前查 `usage`，使用中音檔需明確強制刪除才會執行

### 前端互動細節（除錯過程中修正的問題）

1. **切換分類後選單變灰**：原本切換分類／上傳／刪除後呼叫 `switchPage('sounds')`，會先清空整頁成「載入中」再重繪，閃爍重繪導致 `<select>` 卡住。改為統一呼叫 `renderSoundLibrary()`，不經過 `switchPage`，且不再清空畫面，直接原地更新。

2. **音檔清單被截斷、頁面持續往下延伸**：根因是內建音檔筆數多（1016 筆）一次全部渲染成 DOM，撐大外層容器。解法：
   - 面板改用 `max-height: calc(100vh - 140px)` + flex 排版固定外框高度
   - 清單區域 `flex:1; overflow-y:auto` 在框內捲動
   - 內建音檔分頁載入，預設只渲染 100 筆，捲到底可按「載入更多」

3. **統一播放器按鈕狀態與實際播放不同步**：原邏輯是按鈕自己猜測播放狀態，跟 `<audio>` 元件實際狀態脫鉤。改為：
   - 播放器綁定 `play`/`pause`/`ended` 事件，狀態變化時呼叫 `_soundSyncPlayingBtn()` 依播放器真實狀態重畫所有按鈕
   - 同一首歌再按一次按鈕 = 切換播放／暫停；點別首則換歌重播
   - 不論透過清單按鈕或播放器本身控制列操作，兩邊狀態保持同步

4. **搜尋框輸入時失焦**：原本每打一個字就立即重繪整頁，輸入框被銷毀重建因而失焦。改為 debounce（停止輸入 300ms 後才重繪），並在重繪後自動將焦點與游標還原至搜尋框結尾，銜接使用者繼續輸入。
