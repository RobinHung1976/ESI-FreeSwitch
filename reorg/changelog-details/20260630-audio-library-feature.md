# 音檔庫功能新增 — 2026-06-30

> 原始來源：`audio-library-feature-20260630.md`；現況已併入 `feature-sounds.md`

## 需求

各功能（IVR、分機、語音信箱）音檔各自獨立上傳管理，沒有統一入口瀏覽/試聽/刪除所有自訂音檔，也看不到系統內建可用提示音清單。新增獨立「音檔庫」頁面，IVR 改為共用同一份 API，避免重複實作。

## 除錯過程中修正的問題

1. **切換分類後選單變灰**：原本切換分類/上傳/刪除呼叫 `switchPage('sounds')`，先清空整頁再重繪，閃爍導致 `<select>` 卡住。改為統一呼叫 `renderSoundLibrary()`，不經過 `switchPage`，不清空畫面直接原地更新。
2. **音檔清單被截斷、頁面持續往下延伸**：內建音檔筆數多（實測 1016 筆）一次全部渲染成 DOM 撐大外層容器。解法：面板改 `max-height: calc(100vh - 140px)` + flex 排版固定外框高度，清單區域 `flex:1; overflow-y:auto` 框內捲動，內建音檔分頁載入（預設 100 筆，「載入更多」）。
3. **統一播放器按鈕狀態與實際播放不同步**：原邏輯按鈕自己猜測播放狀態，跟 `<audio>` 元件實際狀態脫鉤。改為播放器綁定 `play`/`pause`/`ended` 事件，狀態變化時呼叫 `_soundSyncPlayingBtn()` 依真實狀態重畫所有按鈕。
4. **搜尋框輸入時失焦**：每打一個字就立即重繪整頁導致輸入框被銷毀重建而失焦。改為 debounce（停止輸入 300ms 後才重繪），重繪後自動將焦點與游標還原至搜尋框結尾。

## 實測結果 ✅

自訂/內建音檔皆可正常試聽、上傳、刪除；分類切換、搜尋、分頁載入運作正常無閃爍或失焦；統一播放器狀態同步正確；刪除使用中音檔會正確攔截並提示，強制刪除流程正常。

詳細 API 設計、安全性機制已完整併入 `feature-sounds.md`。
