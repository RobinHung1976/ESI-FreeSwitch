## ✅ 分機狀態無法即時顯示「通話」「響鈴」（2026-06-26）

**根本原因**：FreeSwitch ESL 事件中的 `Channel-Name` header 值為 `sofia/internal/1126%40192.168.100.209`，`@` 被 URL encode 成 `%40`，導致 `_ext_from_channel_name()` 無法正確解析分機號碼，回傳空字串後 early return，狀態更新完全失效。

**修改檔案**：`server.py` 第 58～72 行

```python
def _ext_from_channel_name(ch_name: str) -> str:
    if not ch_name:
        return ""
    try:
        ch_name = ch_name.replace("%40", "@")  # ← 關鍵修復
        parts = ch_name.split("/")
        if len(parts) < 3:
            return ""
        profile = parts[1].lower()
        if profile not in ("internal", "default"):
            return ""
        num = parts[2].split("@")[0].strip()
        if not num.isdigit():
            return ""
        return num
    except Exception:
        return ""
```

同步修復：
| 問題 | 修復 |
|---|---|
| B leg `CHANNEL_CREATE` 把 `talking` 狀態退回 `ringing` | `CHANNEL_CREATE` 前檢查 `cur.get("status") not in ("talking", "holding")` |
| 分機 offline 後 `CHANNEL_DESTROY` 誤設回 `idle` | `CHANNEL_DESTROY` 前檢查 `cur.get("status") != "offline"` |

## ✅ 分機管理頁面出現異常卡片且無法刪除（2026-06-26）

**現象**：出現 `$$default_pro...` 與 `SEP0011GAABBCC` 兩個異常卡片。

**根本原因**：
- `$$default_pro...`：FreeSwitch 預設 template XML 的 `<user id>` 為空字串，被 `list_extensions()` 掃入清單
- `SEP0011GAABBCC`：Cisco IP Phone 自動建立的設備 XML（`SEP<MAC>.xml`），非分機用途卻被掃入
- `delete_extension()` 用 `ext_id` 組檔名，但這兩個異常 XML 的 `id` 不等於檔名，導致刪除找不到檔案回傳 404

**修復**（`server.py`）：
```python
# list_extensions()
ext_id = user.get('id','').strip()
if not ext_id:
    continue
if not ext_id.isdigit():    # 過濾 SEP...、template 等非數字 id
    continue
# result.append 新增
'filename': os.path.basename(filepath),
# delete_extension() 新增 filename 參數，讓刪除走實際檔名路徑
def delete_extension(ext_id: str, filename: str = Query(None)):
    if filename:
        safe = os.path.basename(filename)
        filepath = f"{EXT_DIR}/{safe}"
    else:
        filepath = f"{EXT_DIR}/{ext_id}.xml"
```
根治邏輯：過濾非純數字的分機 id，讓異常 XML 從源頭就不進入清單。

## ✅ IVR 流程圖預覽：取消時段路由後按鍵節點消失（2026-06-26）

**根本原因**：`_ivrSVGFlowSized()` 的 `layerOf()` 用硬編碼層號，`sched`/`work_t`/`off_t` 不存在時 `key_*` 仍被分配到固定層號 3，造成層與層之間出現空洞，SVG 高度計算錯誤導致節點被裁切。

**修復**：① `layerOf` 改為動態累加建構(依實際存在節點依序分配層號)；② y 座標改用排序後的 `rank`(而非原始層號)確保連續。
